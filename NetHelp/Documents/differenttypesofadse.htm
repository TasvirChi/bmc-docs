Different Types of Ad Servers

There are additional ad servers that Borhan supports, which are a bit more complex than just plugins. For example, Borhan is now fully integrated with DoubleClick and Freewheel. Their integrations are more complex since there is a server-side connector that allows you to define the metadata to submit to the ad server for ad targeting purposes. The Borhan metadata gets mapped to ad serverâ€™s metadata. Then, just like with a distribution module, see Borhan Distribution Module, when a Borhan partner wants an entry to be included in the ad server, it is marked as such in the BMC. The ad server connector passes all of the metadata to the ad server, where the Borhan partner can then decide how ads should be targeted based on the metadata.

Borhan Player Additions Related to Ads

If a Borhan partner wants to create their own type of ad targeting and prevent pre-rolls from playing on specific entries, they can utilize the flashVar vast.preSequence=0, defined in the embed code in the additional parameters and plugins.

Advanced functionality is enabled to provide for a better workflow of ad plugins. The Borhan Player build its UI only when the plugin is completely ready, enabling for a smooth loading of player and plugin together after the plugin is ready.
